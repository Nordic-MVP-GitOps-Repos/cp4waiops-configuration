Check (and if necessary, update) the values for all parameters.

  • The values of SOURCE_RBAHOST and TARGET_RBAHOST depend on the types of the source and target RBA deployments:
    • For RBA Private Deployment: The XX_RBAHOST is the location where the RBA service is running (which may be different from the location where the Netcool OMNIbus/DASH UI server is running) and has the format <fullyQualifiedDomainName>:<rba-rbs-port>, for example, myhost.mycomp.com:3005
    • For RBA within Cloud Event Management on IBM Marketplace: The XX_RBAHOST is for example rba.us-south.cloudeventmanagement.cloud.ibm.com
    • For RBA within Netcool Operations Insight running on Red Hat OpenShift: Then XX_RBAHOST is the location where the NOI service is running, for example, netcool.n163.apps.mycluster.mycompany.com
    • For the complete list of supported deployments and related notes, see the runbook API documentation.
  • The values of SOURCE_APIKEY and TARGET_APIKEY have the format <apiKeyName>:<apiKeyPassword>, for example, ld/abcd:cd+ef.
    • If you do not yet have API keys for the source and target RBA API endpoints, you need to create them first.
    • You need to have the administrative role (e.g., RBA manager) to create API keys.
    • For RBA Private Deployment: Navigate to the "API keys" page.
    • For RBA within Cloud Event Management on IBM Marketplace: See API Keys.
    • For RBA within Netcool Operations Insight running on Red Hat OpenShift: Navigate to "Administration > Integration with other systems" and switch to the "API keys" tab.
    • For more information, see API Keys.
