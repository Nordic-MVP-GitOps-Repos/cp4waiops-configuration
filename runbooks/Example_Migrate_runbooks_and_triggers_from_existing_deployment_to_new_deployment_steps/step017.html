Run the command

export TRIGGER_FILE_NAMES=`ls -v ./triggers/*.json`

to get a list of the file names that have been created in the previous step, and then create the triggers:

for triggerFileName in $TRIGGER_FILE_NAMES; do echo; echo "Starting with ${triggerFileName}"; curlOutput=$(curl -X POST -k -s -u "" -H "Expect:" -H "Content-Type: application/json; charset=utf-8" -d @${triggerFileName} "https:///api/v1/rba/triggers"); if (test ${#curlOutput} -gt 0); then echo ${curlOutput}; else sleep 5; fi; echo "Finished with ${triggerFileName}"; done
 

Check if any errors have occurred while running one or multiple curl POST requests. If needed, you can

  • modify the JSON file(s) for the failing trigger(s) as needed,
  • reset the TRIGGER_FILE_NAMES environment variable to contain these modified JSON files only (including the directory name), for example:
    • export TRIGGER_FILE_NAMES="./triggers/trigger.1.json ./triggers/trigger.5.json"
  • re-run the command.
