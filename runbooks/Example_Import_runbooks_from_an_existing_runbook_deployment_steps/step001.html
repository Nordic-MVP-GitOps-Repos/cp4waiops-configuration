This runbook provides instructions to export runbooks from one deployment and to import them to another deployment. It uses the RBA API for both operations.

  • If you import runbooks more than once, they are duplicated and not replaced.
  • If your runbook refers to a script automation, then the SSH connection must be configured on the target deployment prior to importing the runbook.
  • If several runbooks refer to the same script automation, and these runbooks are exported and imported separately, the automation will be duplicated during import.
  • Exporting multiple runbooks at once will consolidate automations referenced by more than one runbook and only create the minimum number of new automations.
  • The described procedures will not migrate other configuration, such as the following (some of the following configuration features may not be supported by the type of deployment you are using):
    • BigFix automations (because fixlets are stored in BigFix)
    • Ansible automations (because job templates and workflow job templates are stored in Ansible Tower)
    • Triggers (because triggers are stored in Impact)
    • Gateway setup
    • Connections to automation providers
    • Connection to Impact
    • Users and user role mappings
  • The sample "curl" commands in this runbook use the "-k" option to suppress certificate validation. You can adjust this as needed and save a new version of this runbook.
