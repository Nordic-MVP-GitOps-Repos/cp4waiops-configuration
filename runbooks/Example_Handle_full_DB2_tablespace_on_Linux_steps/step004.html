Run the following command to identify the tablespace ID and calculate the current size in bytes of the tablespace on the file system:

 

db2 "select TBSP_ID As \"ID\",TBSP_TOTAL_SIZE_KB * TBSP_TOTAL_PAGES As \"Tablespace size in bytes\" from SYSIBMADM.TBSP_UTILIZATION WHERE TBSP_NAME = '' and TBSP_STATE='NORMAL' and TBSP_TYPE='DMS'"

 

The result should display the tablespace ID and the size of the tablespace, similar to the example below:

db2inst1@db2server:~> db2 "select TBSP_ID As (â€¦)"
ID                       Tablespace size in bytes
------------------------ ------------------------
                       7                 33554432


The example shows the information that the tablespace ID is 7 and that it currently consumes 33,554,432 bytes (32 MiB). Go back to runbook details and add that number as the value of runbook parameter TABLESPACE_ID. Press apply then and proceed with the next step.
