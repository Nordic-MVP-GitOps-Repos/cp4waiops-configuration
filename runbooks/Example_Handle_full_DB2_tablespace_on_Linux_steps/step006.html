Run the following command to determine how much space is available on the file system:

 

df -m

 

The result should look similar to below. We can identify that the /opt file system has 21,094 MiB free space.

Filesystem              1M-blocks      Used Available Use% Mounted on
/dev/mapper/system-root     15320     11386      3155  79% /
/dev/mapper/appvg-apps      61923     37683     21094  65% /opt
/dev/sda1                     154        37       109  26% /boot

Check the  file system and determine if the tablespace can be extended another 10% or not. From previous steps, we know that the tablespace currently consumes 32 MiB. 10% of this is appromately 3.2 MiB, this is lower than the free space on /opt and will therefore fit. If the number is larger than the available space on the file system, this runbook and then escalate to the next support level.
