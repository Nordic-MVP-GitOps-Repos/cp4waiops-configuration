Check whether the itsm.status.external.json kafka topic is populated, by issuing the following command:

oc exec -ti -kafka-0 -- ../../opt/kafka/bin/kafka-console-consumer.sh --from-beginning --bootstrap-server localhost:9092 --topic itsm.status.external.json

If no status messages are there, restart the topology-status service using the following command:

oc delete $(kubectl get pod | grep topology-status | awk '{print $1}')
