Check the eanoiaction service logs, and restart if you believe there should be connectivity to the object server pair but the actions service is not providing it. To check the eanoiactionservice logs use:

oc logs $(oc get pod | grep eanoiaction | awk '{ print $1}')

To restart the eanoiactionservice:

oc delete pod $(oc get pod | grep eanoiaction | awk '{ print $1}')
